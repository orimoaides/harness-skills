# fix-prompt-guide — 画像編集AIに渡す修正プロンプトの作り方

fix_order の各項目を、GPT（gpt-image系）や Nano Banana（Gemini画像編集）等の画像編集AIにそのまま貼れるプロンプトに変換するための知識。評価スキルは直さない — 直すのは外部の生成AIで、ここはその指示書を書く工程。

## 大原則（画像編集AIの特性）

1. **1プロンフト1修正。** 複数の修正を1プロンプトに混ぜると全部が中途半端に動き、無関係な箇所まで動く。fix_orderの項目ごとに別プロンプトを作る。
2. **保持指定が本体。** 編集AIの最大の事故は「頼んでいない所が変わる」こと。変更箇所の指示と同じ分量で**【変更しないもの】リスト**を書く。バナーで必ず保持指定するもの: 全テキストの内容と字形／人物写真／ロゴ／配色／レイアウトの他要素。
3. **日本語文字は再描画させない。** 画像生成AIは日本語テキストの描画が弱く、全体再生成すると文字が崩れる。「文字の再描画はせず、既存の文字要素を保持したまま◯◯のみ調整」を明記する。文字サイズ変更は「拡大」（既存字形のスケール）として指示し、「書き直し」をさせない。
4. **位置とサイズは具体的に。** 「目立たせて」ではなく「約1.3〜1.5倍に拡大」「左下の◯◯ブロックを」。曖昧な形容詞は編集AIの自由裁量＝事故の入口。
5. **意図（なぜ）を1文添える。** 理由があると編集AIは別解を選べる（例: 拡大が構図を壊すなら帯を敷く等）。ただし別解の許容範囲も明示する（「〜しても可」）。
6. **段階指定。** 大きく変えたくないときは「一段」「控えめに」を数値レンジとセットで（「1.5倍以内で」）。

## プロンプトの型

```
添付のバナー画像を編集してください。変更は1点のみ:
【変更箇所】<場所>の<要素>を<具体的な操作・数値レンジ>。
【意図】<なぜ直すか1文。評価の根拠をそのまま使う>
【許容する別解】<あれば。例: 拡大の代わりに背景に薄い帯を敷いて分離しても可>
【変更しないもの】上記以外のすべて — 特に: 全テキストの内容と字形（文字の再描画禁止）／人物写真／ロゴ／配色／他要素のレイアウト。
```

## 運用ルール
- fix_prompts は**編集で解決できるfixにのみ**生成する。検証提案（「縮小表示でテストせよ」等）や人間判断事項（意図宣言・トレードオフ）はプロンプト化しない — 対象外の項目には「検証系のためプロンプト対象外」と明記。
- 改善稿が返ってきたら**同じ評価セットで再評価**し、deltaで「狙ったfixが直ったか・他が後退していないか」を機械比較する。編集AIの副作用（頼んでいない変化）はdeltaのregressedで捕まえる。
- ツール差の目安: 文字保持が重要なバナーでは「既存要素の保持・部分編集」が得意なモデルを推奨（プロンプト内のツール指定はしない — 貼り先は人間が選ぶ）。
- プロンプトは日本語でよい（主要編集AIは日本語指示に対応）。海外モデルで精度が落ちる場合は同じ構造の英訳を併記する。
