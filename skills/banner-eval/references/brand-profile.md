# brand-profile — ブランド差し替え雛形

このパックはブランド非依存。使う組織ごとにこのファイルを複製して値を埋める（`brand-profiles/<ブランド名>.md` として増やしてよい）。埋まっていない項目に依存するA観点は unverified になる（falseにはならない）。

## 1. class A の根拠文書（照合先）
- ロゴレギュレーション: （余白・最小サイズ・改変禁止・使用可能背景。文書パスかURLを書く）
- 入稿規定: （サイズ一覧・容量上限・形式・カラーモード）
- 組版禁則: （行頭禁則文字・日時の行またぎ可否・約物の扱い）
- 表記ルール: （正式名称・肩書きの序列・英数の全半角）

## 2. class B の判断軸
- トーン: （例: 信頼感・親しみ・先進性 のどれを優先するか）
- 配色: （ブランドカラーのhexと使用比率の目安・禁止の組み合わせ）
- タイポ: （使用書体・階層の目安）
- 「らしさ」の言語化: （❌よくあるダメな形 → ✅あるべき形 → 理由、を箇条書きで。レビューで指摘したことを書き溜める）

## 3. お手本ストック
- 良例: （過去案件の最終FIX 2〜3件。何が良いかの1行つき）
- NG例: （初稿→FIXの差分が大きかったもの。何がダメだったかの1行つき）

## 4. 配点の上書き
標準配点（scoring.md）から変えたい場合のみ記述。変えた理由を必ず書く — 理由のない配点変更は較正時に評価できない。
