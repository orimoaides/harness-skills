# class定義 — 観点の三分類

スキル（観点）は必ず A / B / C のどれかに属する。分類の軸は**重要度ではなく測定可能性**。重要度は配点（scoring.md）で表現する。この2軸を混ぜると「大事だけど測れない観点」の扱いを誤る。

## class A — 機械照合・ブロック
- 判定: true/false。照合できるものだけを置く（存在・一致・数値比較）。
- 効力: 1つでもfalseなら差し戻し。意図宣言でも覆らない。
- 変更: 人間のFBでは変えない。規定文書の更新・B観点からの昇格・C への降格の3ルートのみ。
- 典型例: 必須要素の存在（日時・氏名・肩書き・ロゴ）／文字の正確性（原稿との一致）／NG事項・序列制約／改行・禁則（行頭禁則・日時の行またぎ）／出力サイズ・容量・比率／ロゴレギュレーション（余白・最小サイズ・改変禁止）

## class B — 定性判定・10段階採点
- 判定: **10段階スコア**。まず帯を決め（1–3=致命 / 4–7=要改善 / 8–10=基準を満たす）、次に帯内で1点刻み。帯が先の理由: 較正は帯一致で見る（1点単位の一致は人間同士でも無理で、それを求めると較正が永遠に収束しない）。
- 付帯情報: 根拠＋測定手段＋確信度。
- 効力: 配点対象（scoring.md の式で集計）。意図宣言があれば不問にできる（記録は残す）。
- 変更: 較正で動く本体。見落とし2回→基準追加、誤指摘2回→基準を絞る。
- 典型例: 視覚階層（優先順位どおりの目立ち方か）／レイアウト（整列・余白のリズム）／配色（トーン適合・コントラスト）／写真・素材の使い方／訴求・ブリーフ適合／ブランドトーン／技術品質

### B観点の細分化（粒度もハーネスで育てる）
初期のB観点は粗い仮説。目標粒度は「1観点=1判定器」で、そこへは頭でなく較正データで到達する:
1. **細目の芽**: 人間指摘がある観点の特定の側面に集中したら（例: 視覚階層の中の「視線の入り口」だけ2回）、その側面を観点の基準文内に**細目**（箇条書きの判定基準）として追記する。この段階では採点は親観点のまま。
2. **分割（子観点への昇格）**: 細目への指摘・帯ズレがさらに続き、細目ごとに帯が割れる（同じ親の中でpassとfailが混在する）ようになったら、細目を**独立した子観点に分割**する。配点は親の配点を子に割る（合計は不変 — 分割で総点のインフレを起こさない）。
3. **統合（逆方向）**: 分割した子観点の帯が常に一致する（=分ける意味がなかった）なら親に戻す。
- 分割・統合の実行はどちらも人間の承認後。理由: 観点の切り方は「何を別の問題として扱うか」という判断そのもので、ここをAIに任せると観点が指摘のたびに無限増殖する。

## class C — 人間領域（AIは影スコアで挑戦する）
- 判定: **AIも10段階の試行判定（影スコア）を出す。ただし合計点には入れない（集計外）。** あわせて観察事実の列挙＋測定手段＋確信度。
- 理由: 苦手な観点こそ、判定→人間との突き合わせを繰り返すことでしか精度が上がらない。判定を出させないと較正データが永遠に貯まらない。一方で集計に入れると、精度が育つ前の不安定な判定が点数全体の信用を壊す。だから「出すが数えない」。
- 最終判定: 人間。AIの影スコアと観察リストは人間の判定材料として添える。
- 典型例: カーニングの微調整／「らしさ」の最終判断／手作り感・質感の良し悪し
- 昇格の道: 較正で影スコアの**帯一致率が安定**したらBへ昇格（配点対象になる）。一部を機械測定に切り出せたらその部分だけAへ（例: 約物前後の字間の実測）。

## 混在観点の扱い
1つのカテゴリにAとBが混ざるとき（例: 禁則はAだが文字階層はB）は、観点を分割して別IDにする。同一IDに混ぜると、ブロックと減点の境界が曖昧になり意図宣言の効く範囲が不明になる。
