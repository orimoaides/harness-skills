# design-review-panel — 評価し合うモデル

単独のAI評価は「もっともらしいが間違い」を出す。対策は数を増やすことではなく、**視点の違う専門家に独立で評価させてから、互いに反証させる**こと。AI同士は放っておくと同調するので、順序（独立→批判）が本体。

## 前提
- 対象は class B 観点のみ。A は機械照合に議論の余地がなく、討論はトークンの無駄。
- 各専門家は「デザインの一般知識＋自分のフレームワーク」で判定する。判定フォーマットは banner-eval と同一（pass/minor/fail・根拠・測定手段・確信度）。

## 5役（担当フレームワーク）
1. **組版・タイポグラフィ** — 禁則・文字階層・可読性。文字組みの体裁の番人
2. **ブランド監査** — トーン&マナー・ブランドプロファイル適合・凡庸（それっぽいだけ）の検出
3. **アクセシビリティ** — コントラスト・最小文字サイズ・情報の冗長性（色だけに頼っていないか）
4. **視覚階層・構図** — 視線誘導・ゲシュタルト・情報の目立ち順が訴求序列と一致しているか
5. **訴求・ブリーフ適合** — ターゲット適合・訴求優先順位・依頼書との整合

## 3フェーズ

### Phase 1 — 独立評価
各専門家が**他の評価を見ずに**担当観点を判定する。未確認項目も各自申告。
理由: 先に他人の評価を見ると錨づけされ、5役が実質1役になる。

### Phase 2 — 相互批判（反証の義務化）
各専門家は他の4役の評価に対して最低1つ、次のどちらかを試みる:
- **見落としの指摘**: 「あなたの担当領域でここを見ていない」
- **誤指摘への反証**: 「そのfailは根拠が弱い。理由は〜」
「特になし」で済ませない。理由: 反証を任意にすると全員が同調して終わり、討論の意味が消える。反証できなかった場合は「反証を試みたが崩せなかった」と書く — それ自体が判定の頑健性の記録になる。

### Phase 3 — 裁定
裁定役が①合意点（そのまま採用）②乖離点（両論と裁定理由を明記）③人間に委ねる点（確信度が低いまま割れたもの）に分けて統合する。
**乖離は隠さない。** 乖離点こそ較正データの本体 — 人間のFBがどちらに軍配を上げたかで、専門家スキルの基準文が直る。

## 出力
banner-eval のJSONスキーマに `panel` 欄を埋めて返す（`used: true`、`divergence` に乖離点、裁定理由つき）。

## Gotchas
- 毎回フル討論はコストが重い。運用の目安: 較正データが貯まるまでの初期は毎回、以降は「確信度lowが多い」「専門家間乖離が出やすい種類の案件」のみ。
- 5役に同じ画像の同じ欠陥を重複failさせない。観点の担当割り（どの観点はどの専門家の管轄か）を裁定役が先に確認する。重複failは点数を二重に削り、人間採点とのズレを生む。
- 専門家の追加（例: 写真品質）は較正データで「既存5役が拾えない指摘が繰り返し出た」ことを確認してから。役を増やすほど討論コストは非線形に増える。
