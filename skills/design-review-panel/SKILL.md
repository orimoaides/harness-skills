# design-review-panel — 評価し合うモデル

単独のAI評価は「もっともらしいが間違い」を出す。対策は、視点の違う専門家に**本当に独立で**評価させてから互いに反証させること。AI同士は放っておくと同調するので、順序（独立→批判）が本体。

## 5役（担当フレームワーク）
1. **組版・タイポグラフィ** — 禁則・文字階層・可読性
2. **ブランド監査** — トンマナ・プロファイル適合・凡庸（AIっぽさ）の検出
3. **アクセシビリティ** — コントラスト・最小文字サイズ・色だけに頼っていないか
4. **視覚階層・構図** — 視線誘導・ゲシュタルト・目立ち順と訴求序列の一致
5. **訴求・ブリーフ適合** — ターゲット適合・訴求優先順位・依頼書との整合

## 実行方法（重要 — 独立性はサブエージェントで担保する）

### Phase 1 — 独立評価【5体のサブエージェントを1メッセージで並列起動】
1つのコンテキストで5役を順に演じると、後の役に前の役の判定が見えて独立が壊れる。必ず **Agentツールで5役を別々のサブエージェントとして同時に起動**する。各サブエージェントに渡すもの:
- 画像・ブリーフ・自分の担当フレームワークの判定基準・banner-evalと同じ判定フォーマット（10段階=帯1–3/4–7/8–10を先に決め then 帯内1点刻み・根拠・測定手段・確信度）
- **渡さないもの**: 他の役の存在・他の役の判定。担当がclass C観点なら影スコア（集計外）として出させる。未確認項目も申告させる。

### Phase 2 — 相互批判（反証の義務化）
5役の結果が出そろったら、役ごとに「他の4役の評価＋自分のPhase 1の結果」を渡して再度サブエージェントで実行。各役は最低1つ、**見落としの指摘**か**誤指摘への反証**を試みる。「特になし」は禁止 — 反証できなければ「試みたが崩せなかった」と書く（判定の頑健性の記録になる）。

### Phase 3 — 裁定【メインコンテキストで実行】
①合意点（採用）②乖離点（両論＋裁定理由を明記）③人間に委ねる点（低確信のまま割れたもの）に分けて統合する。**乖離は隠さない** — 人間FBがどちらに軍配を上げたかが専門家スキルの基準文を直す最良の較正データ。

## 出力
banner-evalのJSONスキーマの `panel` 欄を埋めて返す（`used: true`、`divergence` に乖離点と裁定理由）。

## Gotchas
- 毎回フル討論はコストが重い。初期（較正データ収集期）は毎回、以降は確信度lowや乖離が出やすい案件のみ。
- 観点の担当割りを裁定役が先に確認し、同じ欠陥の重複failを防ぐ（二重減点は人間採点とのズレを生む）。
- 役の追加（例: 写真品質）は「既存5役が拾えない指摘が繰り返し出た」較正データを根拠にしてから。
