日本語Webサイト・アプリのデザイン品質を、日本語固有のルールで評価・改善するスキル。

英語圏のデフォルト（`line-height: 1.5`、`max-width: 65ch`、Inter 単体指定、フラットな敬語、英語ベースの情報密度）をそのまま日本語に当てると破綻する。Zen はこの破綻を、明文化された判定基準で防ぐ。

## ルーティング

ユーザーの依頼内容と対象から、3つのモードのいずれかを選んで該当する references を Read する。判断に迷う場合は **critique** を選ぶ（横断的に最も安全）。

| モード | 観点 | リファレンス | こういう依頼で選ぶ |
|---|---|---|---|
| **typeset** | 日本語タイポグラフィ | [references/typeset.md](references/typeset.md) | 「フォントが」「行間が」「文字サイズの体系が」「タイポグラフィを直して」「読みにくい」 |
| **clarify** | 日本語UIコピー | [references/clarify.md](references/clarify.md) | 「ボタンのラベルが」「エラーメッセージを」「フォームの文言を」「敬語が揺れてる」「カタカナが多い」 |
| **critique** | サイト全体の横断品質 | [references/critique.md](references/critique.md) | 「全体を見て」「リリース前チェック」「デザインレビュー」「品質を評価」「総合的に」、または対象が特定領域に絞れない場合 |

引数で `--mode typeset` `--mode clarify` `--mode critique` が明示された場合は、その指定に従う。

## 共通の進め方

どのモードでも基本ステップは同じ:

1. **コンテキスト収集** — プロジェクト構造、フレームワーク、CSS手法、フォント設定、対象ファイルを把握する
2. **プロファイル判定** — サイトの用途を `media` / `saas` / `docs` / `dashboard` のいずれかに分類する。プロファイルによって推奨値・壊れやすい場所・優先度が変わる
3. **評価** — 該当する references/*.md に書かれたルールに従って ✅ / ⚠️ / ❌ で判定する
4. **報告** — スコア + 具体的な改善提案を優先度順で出す
5. **修正** — ユーザーが求めた場合、最小限のコード/テキスト変更を提案・実行する

詳細なワークフロー・評価項目・スコア算出式・出力フォーマットは、選んだモードの references/*.md に書かれている。

## やらないこと

- 配色・カラーパレットの評価（Zen の範囲外）
- パフォーマンス最適化（Core Web Vitals 等）
- セキュリティ監査
- ビジネスロジック / API 設計の評価
- マーケティングコピー（キャッチコピー、LPの訴求文）の推敲
- フォントの購入・ライセンス判断（情報提供のみ）
