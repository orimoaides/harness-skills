# critique — 日本語デザイン品質の統合批評モード

`/zen` の統合批評モード。日本語Webサイト・アプリのデザイン品質を横断的に批評する。typeset（タイポグラフィ）と clarify（UIコピー）の観点を統合し、さらにサイト全体の設計品質を独自の観点で評価する。

## ワークフロー

### Step 1: コンテキスト収集

1. プロジェクト構造を把握（フレームワーク、CSS手法、コンポーネント体系）
2. グローバルCSS、フォント設定、テーマ設定を読む
3. 対象が指定されていればそのファイル、なければ以下の優先度で選ぶ:
   - トップページ / ランディングページ
   - 主要な機能ページ
   - フォーム / 入力画面

### Step 2: タイポグラフィ評価

[typeset モードのワークフロー](typeset.md) に従って評価する。判定項目・Reject/Warn 条件・スコア算出式はそちらを一次定義とする（critique 用に再列挙しない）。サブスコアは X/10 で記録。

### Step 3: UIコピー評価

[clarify モードのワークフロー](clarify.md) に従って評価する。判定項目・スコア算出式はそちらを一次定義とする（critique 用に再列挙しない）。サブスコアは Y/10 で記録（`X / N × 10` で 10 点満点換算）。

### Step 4: サイト全体のデザイン品質（critique 独自観点）

まず対象サイトの用途を判定し、プロファイル（media / saas / docs / dashboard）を決める。プロファイルによって壊れやすい場所や注意点が変わる。

以下のリファレンスを読み、日本語サイト全体の品質を評価する:

- [日本語デザイン品質 リファレンス](japanese-design-quality.md)

評価項目:

**情報密度と余白**
- 日本語テキストの視覚密度に対して余白が十分か
- カード内パディングが窮屈でないか

**視覚的階層**
- 見出し・本文・補足が瞬時に区別できるか
- サイズ差やウェイト差が日本語の画数差を考慮しているか

**和洋バランス**
- ナビゲーション、ボタン、コンテンツの言語が一貫しているか
- 英語フォント × 日本語フォントのバランス

**フォーム設計**（フォームがある場合）
- 氏名の順序、住所入力、入力モードが日本語慣習に合っているか

**レスポンシブ対応**
- 見出しやボタンが不自然に折り返していないか
- テーブルが狭い画面で使えるか

**ダークモード**（対応している場合）
- 日本語テキストの可読性が補正されているか

**アクセシビリティ**
- `lang="ja"` の設定
- アイコンボタンの `aria-label`
- テキストのコントラスト比

**壊れやすい場所チェック**（リファレンスのセクション8参照）
- 本文と見出しが同じ line-height / letter-spacing で設計されていないか
- 本文に過剰な letter-spacing が入っていないか
- `word-break: break-all` が全体に適用されていないか
- 表・フォームが本文ルールを引きずっていないか
- mixed-script 見出しが崩れていないか
- URL はみ出し対策が本文を壊していないか

### Step 5: 統合レポート

スコア算出:
- **タイポグラフィサブスコア（X/10）**: typeset モードの算出式をそのまま適用。ベース 10 点 − Reject 1件×3 − Warn 1件×1、下限 1、上限 10。
- **UIコピーサブスコア（Y/10）**: clarify モードの算出式（✅=1 / ⚠️=0.5 / ❌=0、該当要素不在カテゴリは分母から除外）で X / N を出し、`X ÷ N × 10` で 10 点満点に換算（小数点第一位まで）。
- **サイト全体品質サブスコア（W/10）**: 情報密度・視覚的階層・和洋バランス・フォーム設計・レスポンシブ・ダークモード・アクセシビリティ・壊れやすい場所チェック のうち対象サイトに該当する項目だけを評価し、✅=1 / ⚠️=0.5 / ❌=0 で合算、`合計 ÷ 評価項目数 × 10` で換算。
- **総合スコア** = 3 サブスコアの単純平均（小数点第一位まで）。
- **Critical 減点**: `lang` 属性誤り / 日本語フォント未指定 / フォーカスリング全削除 / レスポンシブ完全非対応 / WCAG AA 未達 / Reject 条件 (R1-R6) 2 件以上同時該当 のいずれかが該当する場合、総合スコアから追加で -2 減点（下限 1）。**複数条件が同時該当しても減点は一度だけ（-2 固定、-2×N にはしない）**。該当しない場合は減点なし。

```
## 日本語デザイン品質レビュー

### 総合スコア: X / 10

#### タイポグラフィ: Y / 10
（typeset 観点の要約 — 最大3行）

#### UIコピー: Z / 10
（clarify 観点の要約 — 最大3行）

#### サイト全体の品質: W / 10

✅ 情報密度: ...
⚠️ 和洋バランス: ...
❌ フォーム設計: ...
...

### 改善ロードマップ（優先度順）

1. **[Critical]** ...（ユーザー体験に直結する問題）
2. **[High]** ...
3. **[Medium]** ...
4. **[Nice to have]** ...
```

### Step 6: 修正

ユーザーが修正を求めた場合、優先度の高い項目から具体的な変更を提案・実行する。

## やらないこと

- 機能要件の評価（ビジネスロジック、API設計）
- パフォーマンス最適化（Core Web Vitals 等）
- セキュリティ監査
- マーケティング戦略の評価
