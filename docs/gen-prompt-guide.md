主要な出典: キム氏による落合陽一式AIデザイン術の分解記事（[X, 2026-07-20](https://x.com/kimuai08/status/2079212341250724250)。方法論を本ガイドの言葉で再構成した。**プロンプト原文（画像生成テンプレ・監査ブロック・スライド/Web/LP・6ステップ）は個人利用アーカイブ** `素材/2026-07-24-落合式AIデザイン術-プロンプト原文アーカイブ.md` に保存済み — banner-genの種テンプレとして使う。公開リポジトリには再配布しない）。

**監査ブロックの位置づけ**: 落合式の「別の厳しいCDとして自己監査（10観点×10点・80点未満は修正）」は、**作るAI側の前段チェック**として使う。生成→自己監査で粗を潰す→うちのbanner-eval（独立評価）に出す、の二重構造。自己監査は独立評価の代わりにはならない（作ったAIの自己採点は甘い — 本人も原理7で明言）。

## 制作の骨格 — 一発で完璧を狙わず、回して選ぶ
落合式の本質は「きれいな完成品の一発生成」ではなく5段階の高速回転:
**曖昧な発想を外に出す → 構造化する → 異なる方向で複数試作 → 人間が違和感で選ぶ → 実際の媒体で検証**。
これはうちのループ（brief-dig→探索→生成→評価→較正）と同型 — 生成プロンプトはこの回転の中の1部品であり、単発の魔法ではない。

## 原理（うちの運用に翻訳済み）

### 1. 形容詞ではなくオブジェクトで指示する（最重要）
「高級感」「かっこよく」はAIが勝手に補完して無難な真ん中を返す。**物体とその関係**に置き換える。埋める変数:
主役の物体／周辺の物体／位置関係／状態／表面の質感／光との関係／観察者（視線の動く順）／時間変化（〜する直前・直後）／**禁止する要素**。
→ うちの「❌→✅→理由」形式と同根。画像生成は「国語とロジスティクスの問題」（落合）— センスでなく言語化。

### 2. 写真系はカメラマンの語彙で
被写体・構図・距離・レンズ相当（35/50/85mm）・光（拡散光/逆光/側面光）・素材・**用途**の7変数。
ただし**専門用語の盛りすぎは逆に崩れる** — 結果に効く変数だけ残す（多いほど良いは誤り。うちの「ノイズはコンテキストの敵」と同じ）。

### 3. 「意味的に関連するが物理的に矛盾する」要素を1つだけ
凡庸回避の核。無作為な奇抜さではなく、概念の理解に貢献する矛盾（例: 業務自動化×水路=情報が滞留せず流れる）。**理由を説明できない矛盾は不採用** — 理由型の原則がここにも。

### 4. 誤生成は3分類してから処理する
**破綻**（文字化け・指の崩れ）=修正／**逸脱**（意図と違うが独自性につながる）=検討／**発見**（中心へ昇格できる予想外）=採用。
全部潰すのは素人、発見を選り分けるのがプロ。→ うちの帯判定と同じ「分類が先」の思想。

### 5. 媒体まで設計する
SNSで0.5秒見られるのか、印刷して読むのか、投影か。媒体が変われば余白・文字サイズ・情報量・コントラストの正解が全部変わる。プロンプトに**用途を必ず入れる**。

### 6. 制作と採点を分ける
作ったAIに自己採点させると甘くなる — 別のAI・別の役割で監査する。**うちの分業採点・panelの外部実証**。監査は観点別採点＋「装飾を30%削除しても成立するか」「色をなくしても階層が分かるか」等の破壊質問で行う。

### 7. 却下理由を蓄積する（負の資産が強い）
「良かった案」の保存より「**なぜ却下したか**」の保存が長期的に強い — 成功理由は再現しにくいが、失敗理由は次の生成条件として確実に効く。
→ design-rules.md（❌→✅→理由）・ban-pattern・うちの較正ループの外部実証。却下理由は「もっとかっこよく」でなく「主役が二つあり視線が分散。主役を一つに限定し、二番目は明度とサイズを60%に」のような**次の生成条件になる書式**で書く。

## バナー生成の実務指針（対象タイプ別の文法）

| 対象 | 文法 | 備考 |
|---|---|---|
| 写真系素材 | カメラ語彙（原理2の7変数） | 得意 |
| イラスト素材 | 画風語彙（線・塗り・パレット・参照スタイル） | 得意 |
| 背景・素材パーツ | 用途条件つき（**「コピーを置くため左40%に均質な余白」等、後工程を先に書く**） | 実務の要 |
| 完成バナー一発 | レイアウト＋日本語文字の同時指定 | **最弱。原則避ける（素材分解へ）** |
| 文字 | **生成させない**（「文字・ロゴ・透かしを生成しない」を明記）。文字は編集/組みで後入れ | 定石 |

→ banner-genの本命は「案を素材に分解し、素材ごとに正しい文法でプロンプト化する」変換器。

## 生成時の共通ブロック（型）
1. 用途／伝えたい概念（1文）
2. 主役オブジェクト＋意味ある矛盾1つ（＋なぜ）
3. 構図: 主役の位置・占有率・**後工程用の余白指定**・視線の順
4. 撮影/画風表現（タイプ別文法・効く変数だけ）
5. 素材と光の物理整合（均一なプラスチック質感の禁止）
6. 色: 基調/補助/アクセントの3色制限
7. 品質基準と**避ける表現の列挙**（意味のない発光ライン・粒子・ネオン・回路模様・既視感AIアート）
8. **A/B/C 3方向生成→比較→改善版**（1案一発ではなく方向違いを出させて選ぶ）
9. 文字・ロゴ・透かしの生成禁止

## ミヤマ式（実例系・2026-07取り込み）
ミヤマ氏（[@mmmiyama_D](https://x.com/mmmiyama_D)・8.7万フォロワー・営業部デザイナー）の公開発信から抽出（[構図別プロンプト図鑑](https://note.com/mmmiyama/n/n3debd52d3b94)・[ブランドガイド式](https://note.com/mmmiyama/n/na15d5ca3380b)。全文はメンバーシップ限定¥980/月 — プロンプト原文が要るなら購読は折茂判断）:
1. **「手が止まる」広告の5要素**: 強いコピー／画面に残る日本語タイポ／美人写真で終わらない構図／効いているポイントカラー／**「ふつうの広告っぽさを消すNG指示」**（ネガティブ指定を要素として格上げ — うちの「避ける表現の列挙」と同じ思想を広告文脈で）
2. **構図をテンプレの単位にする** — 万能版／3案同時版／アプリ広告版／構図別（巨大タイポ・雑誌扉・都市型・反復分割）。「対象タイプ」より1段細かい**構図パターン粒度**でプロンプトを図鑑化する発想。banner-genのプリセット設計の参考
3. **ブランドガイドをAIに読ませて世界観を保持させる** — ガイド選択→ChatGPTにアップロード→プロンプト実行→反復指示→次回以降も世界観維持。**brand-profile→生成の接続の実例**（うちのbrand-profileはこの用途にそのまま使える）
4. **日本語文字入りをワンショットで成立させる長文プロンプト**が実用水準にある（本人談「テキストプロンプトのみ&ワンショットで成立させるために長い」。会社のMicrosoft Copilotで商品画像→バナーの実例も）

### ⚠️ 定石の修正（ミヤマ実例による反証）
本ガイド初版の「文字は生成させない（定石）」は**言い過ぎ**だった。2026年時点のGPT系は、強いコピー指定＋タイポ領域確保＋長文ワンショットで**日本語文字入り広告ポスターを実用水準で出せる事例がある**。修正後の指針:
- **SNS投稿・ラフ・社内提案級** → 文字入りワンショットは選択肢（コピー全文をプロンプトに書き、タイポの領域と扱いを指定）
- **入稿級・文字組みが命のバナー** → 依然として文字後入れが安全（禁則・カーニング・原稿一致はAI描画では保証できず、うちのclass A照合を通らない）
- どちらで作ったかを記録し、評価のtypo系観点の帯分布で実力を実測していく（定石も較正対象）

## 変換系（0→1でない場合）の注意
既存画像＋プロンプトでのスタイル変換（例: ミヤマ氏 [@mmmiyama_D](https://x.com/mmmiyama_D) の実例群）は別文法: 元画像が構図と内容を担うため、プロンプトは「スタイル・仕上げの記述」に集中し、保持指定は編集編の原則に従う。0→1用の変数リストをそのまま使わない。
